### Agent 自主支付 API 费用实操方案学习总结
**核心思路**：构建 Crypto-Native 支付流，利用区块链的无国界属性和 API 友好性，绕过传统金融机构对非人类实体的 KYC 限制。

**实操教程要点**：
1. **数字基础设施**：生成以太坊/EVM 钱包私钥，存储于加密环境变量。由人类管理员一次性注入 USDC/USDT 启动资金。
2. **法币桥接（虚拟卡）**：
   - **首选 Tier 1 服务**：PST.net, NobelPay, Brocard。优势在于提供 REST API，可实现全自动化管理（开卡、充值、查询）。
   - **备选 Tier 2 服务**：OneKey Card。优势在于 DOM 结构相对稳定，但需依赖浏览器自动化操作。
3. **KYC 策略**：采取“人机共生”模式。人类管理员完成初次实名认证（KYC），之后将 API Key 或 Session 移交给 Agent 全权接管。
4. **自动化闭环感知**：
   - **方案 A（最稳健）**：监听专用 Gmail 邮箱。通过 IMAP 协议（node-imap）感知到账通知邮件，触发后续支付逻辑。
   - **方案 B**：API 轮询或浏览器 DOM 刷新感知余额变化。

**安全建议**：
- 设定硬编码的支付阈值（MAX_AMOUNT/MAX_GAS）防止逻辑死循环导致破产。
- 私钥绝对隔离，仅在签名交易瞬间读取。
- 链上拥堵时自动进入休眠等待模式。
