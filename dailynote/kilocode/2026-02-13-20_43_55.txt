### VCP 系统中 MCP 工具调用失败问题修复

**问题现象**：
- VCP 日志显示 `未找到名为 "Powershell" 的插件` 和 `未找到名为 "Snapshot" 的插件`
- Agent saxifei 尝试调用 windows-mcp 工具失败

**根本原因**：
1. VCP 插件系统 (`Plugin.js`) 只能识别 VCP 插件目录下的插件，无法直接调用 MCP 服务器
2. `windows-mcp` 是独立的 MCP 服务器进程，不是 VCP 插件
3. Agent 提示词中使用了简化的工具名称（如 `Snapshot`、`Powershell`），这些名称在 VCP 插件系统中不存在

**解决方案**：
1. 启用 MCPO 插件作为 MCP 桥接器（重命名 `plugin-manifest.json.block` 为 `plugin-manifest.json`）
2. 在 MCPO 配置文件中添加 windows-mcp 服务器配置
3. 修改 Agent 提示词，使用 MCPO 调用格式：`tool_name_param: windows-mcp_工具名`

**关键文件**：
- `Plugin/MCPO/plugin-manifest.json` - MCPO 插件配置
- `Plugin/MCPO/custom-mcp-config/mcp-config.json` - MCP 服务器配置
- `Agent/saxifei.txt` - Agent 系统提示词

**工具名称映射**：
- Snapshot → windows-mcp_Snapshot
- Powershell → windows-mcp_Shell
- Click → windows-mcp_Click
- Type → windows-mcp_Type
- Scroll → windows-mcp_Scroll

**注意事项**：
- 修改后必须重启 VCP 服务器才能生效
- MCPO 插件会自动启动 mcpo 服务进程
- 所有 MCP 工具调用都必须通过 MCPO 桥接器
